Redirects

Redirect loop: redirect loop using 301 http responses. Your browser should show an error.
Redirect loop - meta version: redirect loop using <meta http-equiv="refresh">. Your browser will not know what to do.
Redirect chain: 7 redirects using 301 HTTP responses before arriving at the end. Too much for Google.
Meta chain: 7-pages of chain redirects using <meta http-equiv="refresh">. Same as before
Redirects with hsts: 4 redirects, but each time there is a 307 to the https version of the page. Google will call pages 1-3 two times to make sure, then give up.

User-declared Canonicals

Not the index - meta tag Points to "/" as the canonical using a meta tag.
This also is not the index - http header: Points to "/" as the canonical using an HTTP Link: header.

404s

True 404: This link is broken.
Soft 404: Although it returns a 200 http response, will produce a Soft 404 error in Search Console.
Bot 404: This page returns a 404 if you user agent contains "Googlebot", otherwise it returns a 200. It should not be indexed, but you won't see any error in your browser.

Server Errors

The five hundred: Returns a 500 HTTP response (server error).
5xx Robots: A mirror website that prevents crawling by returning a 500 HTTP response on robots.txt.

User directives (robots.txt and noindex)

Blocked: Blocked from crawling by robots.txt.
No index: No-index meta tag.
No index - HTTP: the server returns an http X-Robots-Tag: noindex header.

Rendering

1p Blocked: Products are rendered on the page using javascript, but the script is blocked by robots.txt, so the crawler never sees the full content of the page.
